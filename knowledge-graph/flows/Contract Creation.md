# Contract Creation Flow

## Entry Points

| UI Entry Point | Component |
|---------------|-----------|
| `/contracts` page → "قرارداد جدید" button | `<CreateContractSheet>` (no defaultProjectId) |
| `/projects/[id]` → "قرارداد جدید" button | `<CreateContractSheet defaultProjectId={id}>` |

Both use the same reusable component: `frontend/src/components/domain/CreateContractSheet.tsx`

## Active-Project Gate

**Rule**: Contracts can only be created for projects with `status = "active"`.

Enforcement:
1. **Project Combobox** (when `defaultProjectId` is not set): `ActiveProjectCombobox` queries `GET /projects?status=active`. Only active projects appear in dropdown.
2. **Pre-filled project** (when `defaultProjectId` is set): component fetches the project via `GET /projects/:id`. If `project.status !== "active"`, shows amber warning banner and disables submit button.

_Backend does NOT block this — frontend-only enforcement currently._

## Form Fields

| Field | Notes |
|-------|-------|
| project_id | Active-only combobox OR locked (pre-filled from URL param) |
| contractor_id | Debounced search combobox (`ContractorCombobox`) |
| contract_no | Auto-generated by backend if blank |
| title | Required |
| description | Optional |
| type | `lump_sum` \| `unit_rate` \| `cost_plus` \| `time_material` |
| status | Default `draft` |
| gross_budget + currency | Budget + 3-char currency code |
| starts_on, ends_on | `PersianDatePicker` → outputs ISO `YYYY-MM-DD` |
| retention_pct % | Converted to bps: `Math.round(pct * 100)` |
| advance_pct % | Same |
| vat_pct % | Same |
| social_security_pct % | Same |
| Files (≤3) | Uploaded after contract created (sequential, not atomic) |

## Submission Flow

```
1. POST /contracts  { ...formData, retention_pct_bps: pct*100, ... }
   → returns { data: { id, ... } }
2. for each file in files.slice(0,3):
     POST /contracts/:id/attachments  (multipart)
3. invalidateQueries(["contracts"])
4. toast.success + onSuccess() + onClose()
```

Mutation is in `CreateContractSheet`'s `useMutation`. File upload is sequential (not parallel).  
If file upload fails after contract creation, contract is still saved (no rollback).

## Validation (Zod Schema)

```ts
project_id:    z.string().min(1)
contractor_id: z.string().min(1)
title:         z.string().min(1)
type:          z.enum(["lump_sum","unit_rate","cost_plus","time_material"])
status:        z.enum(["draft","signed","active","closed","cancelled"])
```

All bps/date/budget fields are optional strings.

## Cache Invalidation

On success: `queryClient.invalidateQueries({ queryKey: ["contracts"] })` — prefix match invalidates both `["contracts"]` and `["contracts", projectId]`.
